1. Difference between synchronous and asynchronous file operations

ans. Synchronous (Sync):

Executes line by line
The program waits until the file operation is complete
Example: fs.readFileSync()

“Finish this task first, then move ahead.”

Asynchronous (Async):

Runs in the background
The program does not wait, it continues execution
Example: fs.readFile()

“Start this task and continue; handle the result later.”

2. When should you use file streams?

ans. Use file streams when working with large files.

Reading entire files can consume too much memory 
Streams process data in small chunks 

Real-world examples:

Video streaming
Large log files
Big datasets (CSV, JSON)


3. Purpose of ‘utf8’ encoding

ans. fs.readFile("file.txt", "utf8", ...)
Converts file data into readable text
Without it → you get raw binary (buffer) 
With it → you get proper text 

“utf8 makes file content human-readable.”

4. Common file system error codes
ENOENT → File not found
EACCES → Permission denied
EISDIR → Expected a file but got a directory
ENOTDIR → Expected a directory but got a file

“Error codes help identify exactly what went wrong.”

5. How to safely delete a directory
fs.rm("folder", { recursive: true, force: true }, callback);
recursive: true → deletes all contents inside
force: true → avoids errors if something is missing

Safest way to remove a folder and everything inside it.

6. Concept of piping in streams

ans. Piping means sending data directly from one stream to another.

fs.createReadStream("input.txt")
  .pipe(fs.createWriteStream("output.txt"));

“Like a pipe carrying water from one tank to another.”

7. Why error handling is important

ans. Prevents application crashes
Gives meaningful error messages
Helps in debugging

“Without error handling, even a small issue can break your app.”

8. Difference between writeFile and appendFile

ans. writeFile:

Replaces existing content
“Overwrite everything.”

appendFile:

Adds data to existing content
“Keep old data, just add more.”
