1. What are the advantages of using Mongoose over the native MongoDB driver?

Ans. Mongoose makes MongoDB easier by giving structure (schema), validation, and clean code.
You can define fields, data types, and rules (like age must be >18).
It also supports middleware and relationships (populate), which the native driver doesn’t provide directly.

2. Explain the difference between findOneAndUpdate() and updateOne().

Ans. findOneAndUpdate() updates a document and returns it (very useful when you need updated data).
updateOne() only updates and gives a status result, not the document.

3. What is the purpose of middleware in Mongoose?

Ans. Middleware are functions that run before or after actions like save, update, delete.

Example use:
Hash password before saving
Log activity
Validate data

4. How do you implement pagination in Mongoose?

Ans. Pagination means showing data in parts (pages).

You use:

skip() → skip old data
limit() → number of items per page

5. When should you use embedding vs referencing in MongoDB schema design?

Ans.Embedding means storing data inside the same document.
 Good for small data and fast reading.

Referencing means storing data in another collection and linking with ID.
 Good for large data and scalability.
